    Commands Overview

    DBL provides a comprehensive set of commands organized by functionality. This guide will help you understand what each command does and when to use it.

    Command Categories

    üîß Setup Commands

    Commands for initializing and maintaining your DBL installation.

    Command Description
    dbl init Initialize DBL in your project
    dbl version Show DBL version information
    dbl update Check and install DBL updates

    üèñÔ∏è Sandbox Commands

    Safe testing environment for database changes.

    Command Description
    dbl sandbox start Create an isolated sandbox database
    dbl sandbox apply Apply sandbox changes to main database
    dbl sandbox rollback Discard all sandbox changes
    dbl sandbox status Check sandbox state

    üìù Change Management

    Commands for tracking and committing database changes.

    Command Description
    dbl diff Show changes in sandbox vs baseline
    dbl commit Save changes as a versioned layer
    dbl reset Rebuild database from layers

    üåø Branching

    Git-like branching for parallel development.

    Command Description
    dbl branch List, create, or delete branches
    dbl checkout Switch to a different branch
    dbl merge Merge changes from another branch
    dbl pull Pull changes from another branch
    dbl rebase Rebase current branch onto another

    üìú History & Inspection

    View and validate your database history.

    Command Description
    dbl log View layer history
    dbl rev-parse Resolve references (HEAD, branches)
    dbl validate Validate migration patterns

    Quick Reference

    Common Workflows

    Basic workflow: 

    dbl sandbox start      # Create sandbox
# Make changes in database...
dbl diff              # Review changes
dbl commit -m "msg"   # Save changes
dbl sandbox apply     # Apply to main DB

    Branch workflow: 

    dbl branch feature-x  # Create branch
dbl checkout feature-x # Switch to branch
# Work on feature...
dbl checkout master   # Back to master
dbl merge feature-x   # Merge changes

    Review history: 

    dbl log               # Full history
dbl log --oneline     # Compact view
dbl log -n 5          # Last 5 layers

    Command Syntax Patterns

    DBL follows consistent patterns across commands:

    • Flags: Options starting with - or --
    • Arguments: Required or optional parameters
    • Subcommands: Commands with actions (e.g., sandbox start)

    Examples:

    dbl command [options] [arguments]
dbl command subcommand [options]
dbl command --flag value argument

    Getting Help

    For detailed help on any command: 

    dbl help
dbl <command> --help
